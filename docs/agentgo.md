# 第一章 初识智能体 

## 一、核心定义

智能体 = 任何能够通过**传感器**感知**环境**，并**自主地**通过**执行器**采取**行动**以达成目标的实体。

四个基本要素：环境 → 传感器（感知）→ 自主决策（智能的核心）→ 执行器（行动）→ 反馈环境。

| **P (Performance Measure)** **性能度量** | 评价智能体行为好坏**客观且可量化**的标准       | 回答 **“怎样才算做得好？”** 定义成功的终极指标，例如：安全性、效率、准确性、用户满意度、成本等。 |
| ---------------------------------------- | ---------------------------------------------- | ------------------------------------------------------------ |
| **E (Environment)** **环境**             | 智能体**感知和行动**所处的外部世界             | 包括所有智能体无法完全控制但需与之交互的**各种因素**，如物理世界、其他智能体、数据源等。 |
| **A (Actuators)** **执行器**             | 智能体用来执行动作、对**环境施加影响**的部件   | 是智能体的“手和脚”。例如：机器人手臂、语音输出、API调用等。  |
| **S (Sensors)** **传感器**               | 智能体用来感知环境状态的部件，是获取信息的途径 | 是智能体的“眼睛和耳朵”。例如：摄像头、麦克风、GPS、用户输入接口等。 |

## 二、传统智能体的演进（四种范式）

从简单到复杂、从被动到主动的演进路径：

1. **简单反射智能体**：基于"条件-动作"规则，仅依赖当前感知，无记忆。如恒温器。
2. **基于模型的反射智能体**：引入内部**世界模型**追踪不可直接感知的状态，拥有初级记忆。如隧道中保持对前车位置判断的自动驾驶。
3. **基于目标的智能体**：主动选择能导向特定目标的行动，回答"我应该做什么才能达成目标"。
4. **基于效用的智能体**：在多条可行路径中，选择**效用最大化**的最优方案，回答"哪个选择最好"。

## 三、作为开发者工具的智能体

智能体被深度集成到开发者工作流中，作为辅助工具**增强而非取代**开发者，通过自动化处理繁琐任务，提升开发效率与质量。

## 四、核心概念总结

| 概念          | 说明                                           |
| ------------- | ---------------------------------------------- |
| 自主性        | 智能体"智能"的核心，基于感知和内部状态独立决策 |
| 感知-行动闭环 | 所有智能体行为的基础循环                       |
| 演进逻辑      | 被动反应 → 模型记忆 → 目标驱动 → 效用最优      |





# 2、开发概念

## 1、基础

| 概念      | 它主要回答什么问题          | 在我们工程里的角色                  |
| :-------- | :-------------------------- | :---------------------------------- |
| Rule      | 什么事绝对不能乱来          | 基础规矩、红线、底线                |
| Skill     | 这件事具体应该怎么做        | 固定动作的标准操作手册              |
| Sub Agent | 复杂任务由谁分工处理        | 不同阶段的专业角色                  |
| Workflow  | 这些角色按什么顺序接力      | 前进、暂停、打回、重跑规则          |
| Scripts   | 最后谁来判断到底做没做好    | 统一门禁和事后验证                  |
| MCP       | AI 怎么安全接上外部工程系统 | 外接能力与工具链（含 Unity 等宿主） |

rule：

| 每次代码修改完成后必须编译 | 编译通过后必须跑测试               |
| -------------------------- | ---------------------------------- |
| 测试通过后必须跑事后验证   | 只有三步全部通过，任务才算真的完成 |
